Creating a Repository on GitHUB


1) Create New Repository



Click on the "+" sign on your github account next to "bell" notification icon and then click on create new repository.


2) Naming and scope of Repo



Enter the name of your repository which you want to set and then choose public or private you want to define the scope of repository


Git commands on terminal


1) git init



After creating empty folder in your pc,right click and select open with visual studio (or any other IDE) the create a file. After writing this command in terminal, a new git repository will be created in the folder.


2) git status



"git status" command displays the state of the working directory and the staging area. It lets you see which changes have been staged, which haven't, and which files aren't being tracked by Git.


3) git add .



"git add ." command adds a change in the working directory to the staging area. It tells Git that you want to include updates to a particular file in the next commit.


4) git commit -m "message"



"git commit" command captures a snapshot of the project's currently staged changes. Committed snapshots can be thought of as “safe” versions of a project.


5) git branch -M main



"git branch" command lets you create, list, rename, and delete branches.


6) git remote add origin [URL]



"Copy your repository link from github



Paste that url in URL of command. "git remote add origin" centralises your source code to the other projects. It is developed based on Linux, complete open source and make your code useful to the other git users. We call it as reference. It pushes your code into the Git repository using remote URL of GitHub.


7) git push -u origin main



"git push -u origin main" will push the changes to the main branch.


8) Publishing the page



In order to publish the page, go in settings of your account and select pages, then select the source and branch and click on save. After a couple of minutes the url for visiting your webpage will appear, you can follow it to view the page.
